# MD workflow policy

- Start with the simulation request gate: material system, structure source, force field, boundary conditions, ensemble, timestep, trajectory outputs, and stop criteria must be explicit.
- Use validated local assets, project fixtures, or approved remote execution plans before running or claiming an MD campaign.
- Produce receipt-backed artifacts for generated inputs, run manifests, logs, trajectory summaries, and derived event candidates.
- Escalate missing force-field provenance, unsafe execution paths, non-reproducible seeds, or incomplete trajectory evidence as blockers.
- Hand off only evidence-backed rates, mechanisms, structures, and uncertainty notes to ML, feature-scale, research, or QA agents.
