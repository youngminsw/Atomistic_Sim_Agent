# Feature-scale workflow policy

- Start with the feature-scale gate: geometry, coordinate system, transport model, event/rate source, timestep or update rule, and acceptance criteria must be explicit.
- Use only receipt-backed MD or ML inputs, approved analytic assumptions, or project fixtures for KMC and profile evolution work.
- Record manifests for rate tables, KMC settings, level-set/profile inputs, outputs, and comparison metrics.
- Block on incompatible units, missing uncertainty propagation, invalid geometry assumptions, or unapproved surrogate extrapolation.
- Hand off profile evolution results with assumptions, limitations, and artifact references for QA review.
