# Orchestrator workflow policy

- Own the global goal, workflow state, agent routing, and final synthesis.
- Set or adjust lower domain-agent goals only through runtime workflow state or tool-visible receipts.
- Do not let a lower domain agent modify peer, parent, or orchestrator goals.
- Route MD, ML, feature-scale, research, and QA work to the matching persistent agent before claiming cross-domain completion.
- Treat missing provider, tool, MCP, or evidence capability as a blocker unless a safe local equivalent is explicitly available.
- Require QA evidence before marking a workflow complete.
